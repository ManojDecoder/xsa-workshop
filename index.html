xsa-workshop

XS Advanced version of HANA PM Workshop tools

Maintained by SAP HANA Product Management Contacts: Thomas Jung (@I809764 - thomas.jung@sap.com) Rich Heilman (@I816013 - rich.heilman@sap.com)

From the management website you click create new workshop:

The tool creates an initial exercise with one step in it. You can add more exercises and steps.

You can edit existing exercises or steps by choosing them in the hierarchy.

However nothing gets written back to the server until you press the Save Workshop to Server button

All this tool is really doing is creating a JSON file and saving in the database. You can also upload/download the JSON file for editing locally. This is useful for mass changes or copying of workshops. The json can also be added to the csv file in the project so the content automatically loads upon installation.

This way the complete workshop can be transported with the MTAR, version controlled, etc.

The URL is the same for all the sites (/workshop/admin/ui/exerciseMaster/), you just pass the workshop id as a URL parameter (?workshop=XXX).

You can also use the node key for a particular step in the URL to position right to that template.

In your templates you can use placeholders like . The web page looks for these patterns and takes the group number from the User Id and replaces in-line. Here are the strings it can replace. Let me know if there are others you would need:

var parts = gUserName.split("_");
var grpNumber = parts[1];
if (typeof(grpNumber) != 'undefined' && grpNumber != null){
   myTXT = myTXT.replace(/<group number>/g,grpNumber);
   myTXT = myTXT.replace(/<group_number>/g,grpNumber);  
   myTXT = myTXT.replace(/<group>/g,grpNumber);           
   myTXT = myTXT.replace(/<Your User>/g,gUserName);
   }

This logic does require that you name your users in such a way that they end in _XXX where XXX is your group number. For example DEV162_050.
